## 2. 核心模块设计

### 2.1 无人机动力学模型 (drone_dynamics.py)

#### 状态表示
```python
@dataclass
class DroneState:
    position: np.ndarray      # [x, y, z] 位置 (m)
    velocity: np.ndarray      # [vx, vy, vz] 速度 (m/s)
    acceleration: np.ndarray  # [ax, ay, az] 加速度 (m/s²)
    time: float               # 当前时间 (s)
```

#### 物理约束
| 参数 | 拦截机 | 目标机 |
|------|--------|--------|
| 最大速度 | 35 m/s | 25 m/s |
| 最大加速度 | 12 m/s² | 8 m/s² |
| 最大转弯率 | 2.0 rad/s | 1.2 rad/s |
| 最大爬升率 | 6 m/s | 4 m/s |

#### 动力学更新
```
v_new = v_old + a_cmd × dt
v_new = clamp(v_new, v_min, v_max)  # 速度限幅
p_new = p_old + 0.5 × (v_old + v_new) × dt  # 梯形积分
```

### 2.2 拦截制导算法 (guidance_algorithms.py)

#### 算法对比

| 算法 | 核心公式 | 优势 | 劣势 |
|------|---------|------|------|
| **Pure Pursuit** | a = k × (v_desired - v_current) | 简单可靠 | 轨迹弯曲，效率低 |
| **PN** | a = N × Vc × d(LOS)/dt | 工程成熟 | 对机动目标效果差 |
| **APN** | a = N×Vc×ω + (N/2)×a_target⊥ | 补偿目标加速度 | 估计滞后 |
| **APN-v2** | + 快速估计 + 突发滑行检测 | 适应突发机动 | 计算复杂 |
| **ZEM** | a = N × ZEM / t_go² | 理论最优 | 近距离需增强 |
| **Hybrid** | 自适应切换策略 | 综合优势 | 检测延迟 |

#### ZEM 制导详解

**零脱靶量 (Zero Effort Miss)**:
```
ZEM = r + v_rel × t_go + 0.5 × a_rel × t_go²
```

**制导指令**:
```
a_cmd = N × ZEM / t_go²
```

**关键优势**: 基于位置/速度预测，不依赖目标加速度方向估计，对反预测机动有天然抗性。

### 2.3 逃脱策略算法 (evasion_algorithms.py)

#### 策略分类

| 策略 | 原理 | 对抗目标 |
|------|------|---------|
| **Orthogonal** | 垂直于视线最大加速度 | PN（最大化视线角速度）|
| **Radial** | 沿视线方向速度脉冲 | APN（干扰接近速度估计）|
| **Deceptive** | 假装匀速后突然变向 | 预测型算法 |
| **Anti-Predictive** | 检测追击方加速度反向操作 | APN/Hybrid |
| **High-G Jink** | 持续高G + 随机方向切换 | ZEM（破坏加速度估计）|
| **ZEM Counter** | 近距离突然变向 | ZEM（利用切换延迟）|

#### High-G Jink 详解

**核心机制**:
```python
# 持续最大加速度
accel = max_acceleration

# 随机方向切换（0.5-1.5秒间隔）
if timer > random(0.5, 1.5):
    direction = random_orthogonal_direction()
```

**对抗 ZEM 原理**: ZEM 假设目标在 `t_go` 内保持恒定加速度，High-G Jink 通过持续随机切换使 `a_rel` 估计始终滞后，导致 ZEM 预测位置系统性偏差。

### 2.4 仿真引擎 (simulation_engine.py)

#### 仿真循环
```
while not intercepted and not timeout:
    1. 目标更新状态（逃脱策略计算加速度）
    2. 拦截机计算制导指令
    3. 拦截机更新状态
    4. 检查拦截条件
    5. 记录数据
```

#### 拦截判定
```python
if distance <= intercept_radius:  # 默认 5m
    intercepted = True
```

## 3. 对抗博弈过程

### 3.1 第一轮：基础对抗

| 拦截算法 | 对匀速目标 | 对机动目标 |
|---------|-----------|-----------|
| Pure Pursuit | ✅ 100% | ⚠️ 低效率 |
| PN | ✅ 83% | ❌ 失败 |
| APN | ✅ 67% | ⚠️ 部分成功 |

### 3.2 第二轮：逃脱优化

逃脱方发现：
- PN 依赖视线角速度 → 正交机动最大化角速度
- APN 依赖加速度估计 → 反预测机动反向操作

结果：
- Anti-Predictive 对抗 APN：**100% 逃脱**
- High-G Jink 对抗 ZEM：**100% 逃脱**

### 3.3 第三轮：拦截升级

拦截方发现：
- ZEM 基于位置/速度预测，对反预测机动免疫
- 突发-滑行模式使加速度估计失效

结果：
- ZEM 对抗 Anti-Predictive：**拦截成功**
- ZEM 对抗 High-G Jink：**拦截成功**

### 3.4 最终博弈矩阵

|  | Pure Pursuit | PN | APN | APN-v2 | **ZEM** | Hybrid |
|--|-------------|----|-----|--------|---------|--------|
| **High-G Jink** | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ **逃脱** | ✅ 逃脱 |
| ZEM Counter | ✅ 拦截 | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ **逃脱** | ✅ 逃脱 |
| Orthogonal | ✅ 拦截 | ✅ 逃脱 | ✅ 逃脱 | ❌ 逃脱 | ✅ **拦截** | ❌ 逃脱 |

**结论**: 不存在对所有逃脱策略都 100% 拦截的算法，博弈呈现动态平衡。

## 5. 关键设计决策

### 5.1 为什么选择 ZEM 作为最优拦截算法？

1. **理论基础**: ZEM 基于最优控制理论，最小能量拦截
2. **鲁棒性**: 不依赖目标加速度方向估计，对反预测机动免疫
3. **适应性**: 自然处理机动目标，无需显式补偿

### 5.2 为什么 High-G Jink 能突破 ZEM？

1. **ZEM 假设**: 目标在 `t_go` 内保持恒定加速度
2. **High-G Jink**: 持续随机切换加速度方向
3. **结果**: `a_rel` 估计始终滞后，预测位置系统性偏差

### 5.3 为什么不存在 100% 拦截？

1. **信息不对等**: 逃脱方知道被追，可选择最优策略
2. **物理约束**: 双方都有最大加速度限制
3. **预测困境**: 任何预测算法都对"不可预测"机动敏感
