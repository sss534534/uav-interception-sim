# 无人机拦截仿真系统 - 开发过程记录

## 1. 项目启动

### 1.1 需求分析

**初始需求**: 设计无人机拦截算法，使另一架无人机实施拦截。

**扩展需求**:
1. 实现完整的仿真系统
2. 支持多种拦截算法对比
3. 实现逃脱策略进行对抗
4. 可视化仿真过程
5. 适配真实飞控硬件

### 1.2 技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3 | 快速开发，丰富的科学计算库 |
| 仿真 | 自研 + Gazebo | 灵活控制，支持硬件在环 |
| 可视化 | Matplotlib + Pygame | 静态图表 + 实时动画 |
| 飞控 | PX4 | 开源，工业级，支持加速度指令 |

## 2. 开发阶段

### 阶段一：基础框架搭建

#### 2.1.1 无人机动力学模型

**实现内容**:
- `DroneState` dataclass: 位置、速度、加速度、时间
- `DroneParams`: 物理参数（最大速度、加速度、转弯率等）
- `DroneDynamics`: 状态更新，包含物理约束

**关键设计决策**:
- 使用梯形积分而非欧拉积分，提高精度
- 速度/加速度限幅在更新后执行，确保物理可行性
- 地面约束（z >= 0）防止穿地

**代码量**: ~244 行

#### 2.1.2 目标运动策略

**实现内容**:
- `TargetStrategy` 类
- 三种模式: STRAIGHT（匀速）、SINUSOIDAL（正弦机动）、RANDOM（随机机动）

**问题与解决**:
- 问题: 随机机动初始阶段行为不可预测
- 解决: 添加 `reset()` 方法，确保可重复测试

### 阶段二：制导算法实现

#### 2.2.1 基础算法

**Pure Pursuit（纯追击）**:
```python
# 始终指向目标当前位置
desired_velocity = direction_to_target * max_speed
accel = gain * (desired_velocity - current_velocity)
```

**PN（比例导航）**:
```python
# 加速度与视线角速度成正比
los_rate = (los_unit - prev_los_unit) / dt
accel = N * closing_speed * los_rate
```

**APN（增强型比例导航）**:
```python
# 加入目标加速度补偿
accel = N * Vc * los_rate + (N/2) * a_target_perp
```

#### 2.2.2 升级版算法

**APN-v2**:
- 快速加速度估计（α=0.6）
- 突发-滑行检测
- 自适应导航增益

**ZEM（零脱靶量制导）**:
```python
ZEM = r + v_rel * t_go + 0.5 * a_rel * t_go^2
accel = N * ZEM / t_go^2
```

**关键突破**: ZEM 对反预测机动有天然抗性，因为不依赖目标加速度方向估计。

### 阶段三：逃脱策略开发

#### 2.3.1 基础逃脱策略

| 策略 | 核心思想 | 对抗算法 |
|------|---------|---------|
| Orthogonal | 垂直于视线最大加速度 | PN |
| Radial | 沿视线方向速度脉冲 | APN |
| Deceptive | 假装匀速后突然变向 | 预测型 |

#### 2.3.2 高级逃脱策略

**Anti-Predictive**:
- 检测追击方加速度方向
- 选择反方向机动
- 对抗 APN/Hybrid 有效

**High-G Jink**:
- 持续最大加速度
- 随机方向切换（0.5-1.5秒间隔）
- **突破 ZEM 制导**

**实现细节**:
```python
# 突发-滑行模式
if phase < burst_duration:
    accel = max_acceleration * direction
else:
    accel = max_acceleration * 0.3  # 滑行阶段降低但不归零
```

### 阶段四：仿真与可视化

#### 2.4.1 仿真引擎

**核心功能**:
- 双机状态同步更新
- 拦截判定（距离 < 5m）
- 超时/飞出范围判定
- 数据记录

**问题与解决**:
- 问题: 目标先更新，拦截机用旧状态计算
- 影响: 引入一拍延迟
- 解决: 后续优化为先计算所有指令再同步更新

#### 2.4.2 可视化

**Matplotlib 静态图表**:
- 3D 轨迹图
- 距离-时间曲线
- 速度-时间曲线
- 多算法对比图

**Pygame 实时动画**:
- 3D 投影显示
- 视角旋转/缩放
- 实时状态显示
- 算法切换

## 3. 对抗博弈过程

### 3.1 第一轮：基础对抗

**测试场景**: 匀速直线目标

| 算法 | 拦截率 | 问题 |
|------|--------|------|
| Pure Pursuit | 100% | 轨迹弯曲 |
| PN | 83% | 对机动目标失效 |
| APN | 67% | 加速度估计滞后 |

### 3.2 第二轮：逃脱优化

**逃脱方策略**:
- 正交机动 → 突破 PN
- 反预测机动 → 突破 APN

**结果**:
- Anti-Predictive 对抗 APN: **100% 逃脱**
- High-G Jink 对抗 ZEM: **100% 逃脱**

### 3.3 第三轮：拦截升级

**拦截方策略**:
- ZEM 制导 → 对反预测机动免疫
- 突发-滑行检测 → 适应机动目标

**结果**:
- ZEM 对抗 Anti-Predictive: **拦截成功**
- ZEM 对抗 High-G Jink: **拦截成功**

### 3.4 最终博弈矩阵

| 逃脱策略 | Pure Pursuit | PN | APN | APN-v2 | **ZEM** | Hybrid |
|---------|-------------|----|-----|--------|---------|--------|
| **High-G Jink** | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ **逃脱** | ✅ 逃脱 |
| ZEM Counter | ✅ 拦截 | ✅ 逃脱 | ✅ 逃脱 | ✅ 逃脱 | ✅ **逃脱** | ✅ 逃脱 |
| Orthogonal | ✅ 拦截 | ✅ 逃脱 | ✅ 逃脱 | ❌ 逃脱 | ✅ **拦截** | ❌ 逃脱 |

**结论**: 不存在对所有逃脱策略都 100% 拦截的算法。

## 4. 关键问题与解决

### 4.1 尾追场景 PN/APN 失败

**问题**: 拦截机初始速度与目标同向，视线角速度为零，无指令输出。

**根因**: Bootstrap 条件仅检查速度大小，未检查视线角速度是否有效。

**解决**: 修改 Bootstrap 条件，检查 `_prev_los_unit is None`。

### 4.2 实时动画速度倍率失效

**问题**: `int(1.5) = 1`，非整数倍速无效果。

**解决**: 使用累积时间方式实现亚整数倍速。

### 4.3 仿真引擎一拍延迟

**问题**: 目标先更新，拦截机用旧状态计算指令。

**影响**: 系统性偏差。

**解决**: 先计算所有指令，再同步更新状态。

### 4.4 ZEM 制导近距离发散

**问题**: 近距离 `t_go` 很小，ZEM 指令过大。

**解决**: 加入 PN 增强项，近距离混合 ZEM + PN。

## 8. 经验总结

### 8.1 设计经验

1. **模块化设计**: 动力学/制导/逃脱/仿真分离，便于独立测试
2. **抽象基类**: `EvasionStrategy` / `GuidanceAlgorithm` 统一接口
3. **双模式支持**: 仿真模式快速迭代，SITL 模式验证真实场景

### 8.2 开发经验

1. **先仿真后实机**: 降低调试成本
2. **对抗迭代**: 拦截/逃脱交替优化，发现算法弱点
3. **可视化**: 动画帮助理解算法行为

### 8.3 算法经验

1. **ZEM 制导**: 理论最优，但需近距离增强
2. **High-G Jink**: 简单但有效，突破 ZEM
3. **混合策略**: 自适应切换，但检测延迟是瓶颈
